Identificar los Frameworks CSS



Tema 2: Frameworks CSS




Los Frameworks son aplicaciones que se construyen a partir de patrones que utilizan el lenguaje orientado a objetos, estos se pueden personalizar e intercambiar en el desarrollo de las páginas web o aplicaciones para que estas queden funcionando correctamente, además permite utilizar menos tiempo y ser mas competitivas cuando se está realizando dichas páginas o aplicaciones. Por lo anterior los Frameworks buscan reducir procesos, utilizar códigos, aligerar procesos, utilizar buenas prácticas, disminuir tiempo en la construcción de páginas o aplicaciones, entre otros. Existen Frameworks para diversos lenguajes como JavaScript, Html5 y CSS. Para el caso del tema que se está trabajando las Frameworks CSS se crean para facilitar el desarrollo de las aplicaciones. Existen ejemplos de Frameworks CSS como 960 Grid System, Blueprint CSS, Simple, Blueprint, YUI Grids CSS, Tripoli,Boilerplate, Blue Trip, entre otros.


Historia de los Frameworks CSS


Los Frameworks comenzaron a utilizarsw en la construcción de apliaciones pequeñas con el Web 2.0.


Frameworks más utilizado


Los Frameworks que se utilizam mas en CSS son:
1. 960 Grid System: Se creo para tener un estádar en el ancho de las páginas web, lo que permitio facilitar la maquetación y confiuración de las columnas. Se caracterísa por: un diseño basado en retícula, compatibilidad con los navegdores, estrutura básica para el diseño, mayor velocidad, entre otras. Sus desventajas: estrutura rígida, limita la creatividad, no se adapta a resolucions sueriores a 1024 px, dificil de impemntar.
2. Blueprint CSS: Se creo como apoyo del diseño y maquetción de la Web 2.0. Se caracteriza por una rejilla que crea cualquier estrutura de la página web, tipografía definicda, estilos para los forularios, impresiones, Intrnet explorer, entre otros. Sus desventajas se percibe como un sistema rígido, coloca mayor atención a la estructura que al contenido, para utilizarse se debe tener conocimientos en CSS.
